    移块上云插件文档

    1. 简介

    本项目为多包协同开发工程，用于各个package的开发、调试、发布。

    项目采用了 monorepo 架构，packages 下的各个模块，会在开发阶段基于 pnpm workspace 链接到 apps 下的各个工程内；

    在构建阶段，可以选择将 packages 下指定模块，发布到 npm 仓库中（使用公司私仓发布）。

    项目结构如下

    .
├── apps                                 # 工程目录，用于调试
│   └── test-example
│       ├── ...
│       ├── package.json

    • node >= 20.19.6
    • npm >= 10.8.2

    3. 快速开始

    3.1 安装依赖

    # 根目录

    pnpm install

    3.2 开发调试

    微信小程序不会直接使用 node_modules，而是执行一次构建npm后，构建并使用 miniprogram_npm 下的文件。

    因此为了实现各个插件 package 的重载，需要在 build-dev 后执行一次 构建NPM操作。

    该操作已经集成到指令 reload 中。

    # 调试目标模块（以 plugin-websocket 为例）

    npm run reload webSocket

    通常指令执行完毕后，微信开发者工具会自动重新加载，页面会直接更新；

    如果页面未正常更新，则需要手动在微信开发者工具执行【清除缓存】操作；

    个别极端情况，如构建文件数过多或构建文件体积较大，需要重启微信开发者工具。

    3.3 发布

    以 plugin-websocket-ws 为例

    发布前请确认库名和版本号正确

    # 预发（内部调试阶段使用，正式上线可省略此步骤）
    npm run webSocket:prerelease

    # 构建目标模块（如果已构建，可省略此步骤）
    npm run webSocket:build

    # 发布
    npm run webSocket:publish

    6. 指令详情

    # 构建
    npm run build ai
    npm run build-dev ai

    # 预发布
    npm run prerelease ai

    # 发布（带确认，自动生成标签）
    npm run publish ai
    npm run publish ai --dev
    npm run publish ai --skip-confirm

    # 快速发布（无确认）
    npm run publish:quick ai

    # 基本使用 - 开发热重载
    npm run reload ai
    npm run reload dcn
    npm run reload scene

    # 带参数
    npm run reload ai --force      # 强制重新构建
    npm run reload ai --skip-build # 只执行小程序 CI，不构建
    npm run reload ai --help       # 查看帮助

    # 发布的包: api-dcn，版本: 0.0.5
    # 生成的标签: api-dcn-0.0.5
